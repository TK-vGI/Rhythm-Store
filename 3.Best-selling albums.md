# Stage 3/6: Best-selling albums
## Description
Add a way to select the statistics. This requires a query to list the name and number of sales of the five best-selling albums. Can you handle it?

Objectives
- You should find the **best-selling** albums. Album sales are the total sales of the music tracks in it.  
- The output should include the album `title` and the _number of sales_.  
- The output should be sorted by the _number of album sales_ in **descending** order, with only the top five selling albums.  
- The column for the _number of album sales_ must be set to `number_of_album`!

Take a look at the database outline below:

## Explanation of the database

![Explanation of the database](./IMG1.png)

The ` **Album** ` table:  
- `AlbumId`: INTEGER Primary key,
- `Title`: TEXT
- `ArtistId`: INTEGER Foreign key from `Artist` table

The ` **Artist** ` table:  
- `ArtistId`: INTEGER Primary key,
- `Name`: TEXT 

The ` **Playlist** ` table:  
- `PlaylistId`: INTEGER Primary key,
- `Name`: TEXT

The ` **Genre** ` table:  
- `GenreID`: INTEGER Primary key
- `Name` : TEXT

The ` **MediaType** ` table:  
- `MediaTypeID`: INTEGER Primary key
- `Name` : TEXT

The ` **Track** ` table:  
- `TrackId`: INTEGER Primary key
- `Name`: TEXT,
- `AlbumId`: INTEGER Foreign key from `Album` table
- `MediaTypeID`: INTEGER Foreign key from `MediaType` table
- `GenreID`: INTEGER Foreign key from `Genre` table
- `Composer`: TEXT
- `Miliseconds`: INTEGER
- `Bytes`: INTEGER
- `UnitPrice`: NUMERIC

The ` **PlaylistTrack** ` table:  
- `PlaylistId`: INTEGER Foreign key from `Playlist` table,
- `TrackId`: INTEGER Foreign key from `Track` table 

The ` **Customer** ` table:  
- `CustomerId`: INTEGER Primary key
- `FirstName` : TEXT
- `LastName` : TEXT
- `Company` : TEXT
- `Address` : TEXT
- `City` : TEXT
- `State` : TEXT
- `Country` : TEXT
- `PostalCode` : TEXT
- `Phone` : TEXT
- `Fax` : TEXT
- `Email` : TEXT
- `SupportRepId` : INTEGER

The ` **Invoice** ` table:  
- `InvoiceId`: INTEGER Primary key
- `CustomerId`: INTEGER Foreign key from `Customer` table
- `InvoiceDate` : DATETIME
- `BillingAddress` : TEXT
- `BillingCity` : TEXT
- `BillingState` : TEXT
- `BillingCountry` : TEXT
- `BillingPostalCode` : TEXT
- `Total` : NUMERIC

- The ` **InvoiceLine** ` table:  
- `InvoiceLineId`: INTEGER Primary key,
- `InvoiceId`: INTEGER Foreign key from `Invoice` table
- `TrackId`: INTEGER Foreign key from `Track` table
- `UnitPrice`: NUMERIC Foreign key from `Track` table
- `Quantity`: INTEGER

The ` **Employee** ` table:  
- `EmployeeId`: INTEGER Primary key
- `LastName` : TEXT
- `FirstName` : TEXT
- `Title` : TEXT
- `ReportsTo` : INT
- `BirthDate` : DATETIME
- `HireDate` : DATETIME
- `Company` : TEXT
- `Address` : TEXT
- `City` : TEXT
- `State` : TEXT
- `Country` : TEXT
- `PostalCode` : TEXT
- `Phone` : TEXT
- `Fax` : TEXT
- `Email` : TEXT

The sales are located in the `Invoice` table. Details of sales are displayed in the `InvoiceLine` table, line by line.  
Music tracks are located in the `Track` table. Albums are found in the `Album` table. There is a relationship between  
music pieces and albums and between music pieces and sales details.
